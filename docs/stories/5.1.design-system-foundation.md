# Story 5.1: Setup shadcn/ui Design System Foundation

## Epic
Epic 5: UI Redesign with shadcn/ui Design System

## Status
Ready for Review

## Story

As a developer,
I want to set up the shadcn/ui design system foundation with Tailwind CSS configuration,
So that I can build consistent, accessible UI components following the approved design specification.

## Story Context

**Existing System Integration:**
- Integrates with: Next.js 16 App Router, existing Tailwind CSS setup
- Technology: shadcn/ui components, Tailwind CSS, Lucide React icons, Inter font
- Follows pattern: Component-based architecture with TypeScript
- Touch points: `tailwind.config.ts`, `src/app/globals.css`, `src/app/layout.tsx`, new `src/components/ui/` directory

**Design Reference:**
- Specification: `docs/front-end-spec.md`
- Implementation guide: `docs/IMPLEMENTATION_GUIDE.md`

## Technical Notes

### Installation Approach

```bash
# 1. Install shadcn/ui
npx shadcn-ui@latest init

# 2. Add core components
npx shadcn-ui@latest add button card badge alert skeleton separator

# 3. Add dependencies
npm install lucide-react date-fns
```

### Key Files to Modify

1. **tailwind.config.ts** - Add brand colors and font configuration
2. **src/app/globals.css** - Add CSS variables and custom classes
3. **src/app/layout.tsx** - Add Inter font integration
4. **src/components/ui/** - shadcn/ui components (auto-generated)
5. **src/components/dashboard/Breadcrumbs.tsx** - New component
